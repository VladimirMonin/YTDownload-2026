## 1. Первый запуск

1. Распакуйте архив в любую папку, например `C:\Programs\YTDownloader\`.
2. Запустите `YTDownloader.exe`.
3. При первом запуске приложение автоматически создаст:
   - Папку для логов: `%APPDATA%\YTDownloader\logs\`
   - Файл настроек: `%APPDATA%\YTDownloader\settings.json`
   - Файл истории: `%APPDATA%\YTDownloader\history.json`
4. Главное окно откроется сразу — никаких установщиков и регистрации не требуется.

> **Антивирус.** При первом запуске Windows SmartScreen или антивирус может показать
> предупреждение о неизвестном издателе. Нажмите «Подробнее» → «Всё равно запустить».
> Это штатное поведение для подписанных EXE-файлов без коммерческого сертификата.

## 2. Настройка приложения

Откройте настройки через кнопку шестерёнки (⚙) в правом верхнем углу или через меню.

### 2.1 Выходная папка

Это папка, куда будут сохраняться все загруженные файлы.

- По умолчанию: `%USERPROFILE%\Downloads\YTDownloader\`
- Нажмите кнопку **«Обзор»** и выберите любую удобную папку.
- Папка будет создана автоматически, если не существует.

> Внутри выходной папки приложение создаёт подпапки по имени видео или плейлиста —
> файлы никогда не смешиваются в одной директории.

### 2.2 Прокси

Если доступ к YouTube заблокирован или требуется маршрутизация через конкретный сервер:

1. В поле **«Прокси»** введите адрес в одном из форматов:

   | Протокол | Пример |
   |---|---|
   | SOCKS5 | `socks5://127.0.0.1:1080` |
   | SOCKS5 с паролем | `socks5://user:pass@127.0.0.1:1080` |
   | HTTP | `http://127.0.0.1:8080` |
   | HTTPS | `https://proxy.example.com:3128` |

2. Нажмите **«Сохранить»**. Приложение проверит формат и предупредит об ошибке.
3. Прокси применяется **ко всем последующим загрузкам**, в том числе к запросам метаданных.

> Чтобы отключить прокси — очистите поле и сохраните настройки.

### 2.3 Порт MCP-сервера

По умолчанию встроенный MCP-сервер запускается на порту **8765**.

Если этот порт занят другой программой:

1. В поле **«MCP порт»** укажите любой свободный порт от `1024` до `65535`.
2. Сохраните настройки и **перезапустите приложение**.
3. Обновите конфигурацию MCP-клиента (Claude Desktop, VS Code) — адрес изменится на новый порт.

## 3. Загрузка видео

1. Скопируйте ссылку на видео из браузера.  
   Поддерживаются форматы:
   - `https://www.youtube.com/watch?v=XXXXXXXXXXX`
   - `https://youtu.be/XXXXXXXXXXX`
   - `https://www.youtube.com/watch?v=XXXXXXXXXXX&t=120` (с таймстемпом — игнорируется)

2. Вставьте ссылку в поле ввода вверху окна.

3. При необходимости измените [опции загрузки](#5-опции-загрузки) (качество, субтитры и т.д.).

4. Нажмите кнопку **«Добавить»** или клавишу `Enter`.

5. В очереди появится карточка загрузки:
   - Сначала отображается URL — приложение запрашивает метаданные.
   - Через несколько секунд название видео обновится на настоящее.
   - Полоса прогресса показывает текущий прогресс загрузки.
   - Статус **«Слияние»** означает, что FFmpeg объединяет видео и аудио-дорожки.

> **Важно.** Если ссылка содержит и видео, и параметр плейлиста
> (`watch?v=XXX&list=YYY`), приложение **всегда загрузит только одно видео**.
> Для загрузки всего плейлиста используйте ссылку вида `/playlist?list=YYY`.

## 4. Загрузка плейлистов

1. Откройте плейлист на YouTube.
2. Скопируйте ссылку из адресной строки:
   - `https://www.youtube.com/playlist?list=XXXXXXXXXXXXXXXXXXXXXXXXXX`
3. Вставьте ссылку и нажмите **«Добавить»**.
4. Приложение загрузит все видео плейлиста последовательно (по одному).
5. Для каждого видео создаётся отдельная подпапка внутри папки плейлиста.

> Если во время загрузки одно из видео недоступно (удалено, приватное),
> приложение пропустит его и продолжит с остальными.

## 5. Опции загрузки

Опции устанавливаются **до добавления** ссылки в очередь. Их можно менять глобально
в настройках или локально в панели опций под полем ввода.

### 5.1 Качество видео

| Значение | Описание |
|---|---|
| **Наилучшее** (по умолчанию) | Максимальное доступное качество |
| **1080p** | Full HD |
| **720p** | HD |
| **480p** | Стандартное качество |
| **360p** | Низкое качество, экономия трафика |

Если запрошенное качество недоступно для конкретного видео — будет выбрано
ближайшее более низкое.

### 5.2 Только аудио

Включите переключатель **«Только аудио»**, чтобы сохранить только звуковую дорожку
в формате `.m4a`. Видео не скачивается и не сохраняется.

### 5.3 Субтитры

1. Включите флаг **«Субтитры»**.
2. Укажите код языка в поле рядом, например:
   - `ru` — русский
   - `en` — английский
   - `kk` — казахский
   - `uk` — украинский
3. Субтитры сохраняются в папке видео в формате `.vtt`.

> Субтитры доступны только если YouTube предоставляет их для конкретного видео
> или плейлиста. Автосгенерированные субтитры YouTube также поддерживаются.

### 5.4 Описание и превью

- **Описание** — сохраняет полный текст описания видео с YouTube в файл `.description`.
- **Превью (thumbnail)** — сохраняет обложку видео в формате `.webp` или `.jpg`.

Оба параметра независимы друг от друга и могут быть включены одновременно.

## 6. История загрузок

Вкладка **«История»** показывает все прошлые загрузки — завершённые, неудавшиеся и отменённые.

- Нажмите на запись, чтобы увидеть путь к файлам.
- Кнопка **«Открыть папку»** откроет директорию с файлами в Проводнике.
- Кнопка **«Удалить»** удалит запись из истории (файлы на диске остаются).

История хранится в файле `%APPDATA%\YTDownloader\history.json`.

## 7. Структура файлов на диске

```
Выходная папка/
│
├── Название видео/              ← одиночное видео
│   ├── video.mp4
│   ├── thumbnail.webp
│   ├── ru.vtt                   ← субтитры (если запрошены)
│   ├── description.txt          ← описание (если запрошено)
│   └── info.json                ← метаданные yt-dlp
│
└── Название плейлиста/          ← плейлист
    ├── 001 - Название видео 1/
    │   ├── video.mp4
    │   └── ru.vtt
    ├── 002 - Название видео 2/
    │   └── video.mp4
    └── 003 - Название видео 3/
        └── video.mp4
```

## 8. Model Context Protocol (MCP)

### 8.1 Что такое MCP

YTDownload запускает встроенный **MCP-сервер** — это позволяет AI-ассистентам
(Claude Desktop, GitHub Copilot, Cursor и другим) напрямую управлять загрузками
через естественный язык.

Пока YTDownloader запущен, MCP-сервер работает на:

```
http://localhost:8765/mcp
```

> Порт `8765` можно изменить в настройках (см. [раздел 2.3](#23-порт-mcp-сервера)).

**Примеры команд в чате AI:**

- *«Скачай это видео: https://youtu.be/dQw4w9WgXcQ»*
- *«Покажи все завершённые загрузки»*
- *«Найди всё, что я скачивал по теме Python»*
- *«Покажи транскрипцию загрузки №5»*
- *«Где лежат файлы загрузки №3?»*
- *«Прочитай описание загрузки №12»*

---

### 8.2 Подключение к Claude Desktop

1. Откройте файл конфигурации Claude Desktop:
   - Windows: `%APPDATA%\Claude\claude_desktop_config.json`

2. Добавьте блок `mcpServers`:

```json
{
  "mcpServers": {
    "ytdownloader": {
      "url": "http://localhost:8765/mcp"
    }
  }
}
```

3. Перезапустите Claude Desktop.
4. В чате должна появиться иконка инструментов — MCP подключён.

> YTDownloader должен быть **запущен** во время работы с Claude.

---

### 8.3 Подключение к VS Code Copilot

1. Откройте настройки VS Code (`Ctrl+,`) и найдите раздел **MCP**.
2. Или добавьте в `.vscode/mcp.json` в рабочем пространстве:

```json
{
  "servers": {
    "ytdownloader": {
      "type": "http",
      "url": "http://localhost:8765/mcp"
    }
  }
}
```

3. Используйте режим **Agent** в Copilot Chat — инструменты станут доступны автоматически.

### 8.4 Справочник инструментов

Все инструменты MCP доступны без дополнительной аутентификации пока приложение запущено.

---

#### `add_download` — Добавить загрузку

Ставит видео или плейлист в очередь загрузки.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `url` | string | — | Ссылка на YouTube-видео или плейлист |
| `quality` | string | `"best"` | Качество: `best`, `1080p`, `720p`, `480p`, `360p`, `audio` |
| `download_type` | string | `"video"` | `"video"` — видео+аудио в mp4, `"audio"` — только аудио в m4a |
| `subtitle_lang` | string | `null` | Код языка субтитров (`ru`, `en`, `kk`) |
| `save_subtitles` | bool | `false` | Сохранять файл субтитров |
| `save_description` | bool | `false` | Сохранять текст описания |
| `save_thumbnail` | bool | `false` | Сохранять обложку видео |

**Возвращает:**
```json
{ "task_id": 42, "status": "queued" }
```

**Примеры:**
```
Скачай это видео: https://youtu.be/dQw4w9WgXcQ
Скачай плейлист https://youtube.com/playlist?list=... с субтитрами на русском
Скачай аудио с https://youtu.be/... в лучшем качестве
```

---

#### `list_downloads` — Список загрузок

Возвращает историю загрузок с фильтрацией по статусу.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `status` | string | `null` | Фильтр: `queued`, `downloading`, `merging`, `done`, `failed`, `cancelled` |
| `limit` | int | `50` | Максимум записей (макс. 200) |

**Примеры:**
```
Покажи все загрузки
Покажи последние 10 завершённых загрузок
Что сейчас скачивается?
Какие загрузки завершились ошибкой?
```

---

#### `get_download` — Детали загрузки

Полные данные одной загрузки включая **абсолютные пути** ко всем файлам.

**Параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `id` | int | Числовой ID загрузки |

**Возвращает:** *(полный формат)*
```json
{
  "id": 42,
  "title": "Rick Astley - Never Gonna Give You Up",
  "status": "done",
  "quality": "1080p",
  "video_path": "C:\\Downloads\\YTDownloader\\Rick Astley\\video.mp4",
  "audio_path": null,
  "subtitle_paths": ["C:\\Downloads\\YTDownloader\\Rick Astley\\en.vtt"],
  "description_path": "C:\\Downloads\\YTDownloader\\Rick Astley\\description.txt",
  "thumbnail_path": "C:\\Downloads\\YTDownloader\\Rick Astley\\thumbnail.webp",
  "info_json_path": "C:\\Downloads\\YTDownloader\\Rick Astley\\info.json",
  "output_dir": "C:\\Downloads\\YTDownloader\\Rick Astley"
}
```

---

#### `search_downloads` — Поиск по истории

Ищет загрузки по подстроке в названии или URL (без учёта регистра).

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `query` | string | — | Строка поиска |
| `limit` | int | `20` | Максимум результатов (макс. 100) |

**Примеры:**
```
Найди всё, что я скачивал про Python
Я скачивал что-нибудь от MongoDB?
Найди загрузки с этого плейлиста: youtube.com/playlist?list=...
```

---

#### `get_file_paths` — Пути к файлам

Возвращает абсолютные пути ко всем файлам конкретной загрузки.

**Параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `id` | int | Числовой ID загрузки |

**Возвращает:**
- `video_path` — путь к `.mp4`
- `audio_path` — путь к `.m4a` (если режим аудио)
- `subtitle_paths` — список путей к `.vtt` / `.srt`
- `description_path` — путь к `.description`
- `thumbnail_path` — путь к `.webp` / `.jpg`
- `info_json_path` — путь к `info.json` yt-dlp
- `output_dir` — папка загрузки

**Примеры:**
```
Где лежат файлы загрузки №5?
Дай путь к субтитрам загрузки №10
Открой папку с загрузкой №3
```

---

#### `get_transcript` — Транскрипция / субтитры

Читает содержимое файла субтитров и возвращает текст.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `id` | int | — | ID загрузки |
| `lang` | string | `null` | Язык: `ru`, `en`, `kk`, `en-orig`. Пусто = первый доступный |
| `timestamps` | bool | `false` | Добавить тайм-коды `[ЧЧ:ММ:СС]` перед каждой фразой |
| `raw` | bool | `false` | Вернуть сырой VTT/SRT с разметкой |

**Режимы вывода:**

- `raw=false, timestamps=false` — чистый текст без разметки *(по умолчанию)*
- `raw=false, timestamps=true` — текст с тайм-кодами: `[00:03:42] Вот пример`
- `raw=true` — исходный VTT/SRT со всей разметкой

**Примеры:**
```
Покажи транскрипцию загрузки №5
Сделай краткое изложение видео из загрузки №7
На какой минуте говорили про архитектуру в загрузке №12? (с тайм-кодами)
Переведи субтитры загрузки №3 на английский
```

> Субтитры доступны только если при загрузке был включён флаг «Субтитры».

---

#### `read_description` — Описание видео

Читает полный текст описания видео, сохранённого при загрузке.

**Параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `id` | int | ID загрузки |

**Возвращает** текст описания (до 64 КБ).

**Примеры:**
```
О чём это видео? (загрузка №5)
Покажи описание плейлиста №12
Что написано в описании курса из загрузки №3?
```

> Описание доступно только если при загрузке был включён флаг «Описание».

---

#### `cancel_download` — Отменить загрузку

Останавливает активную или ожидающую загрузку. Требует подтверждения (двухфазный процесс).

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `id` | int | — | ID загрузки |
| `confirm` | bool | `false` | `false` — предпросмотр, `true` — реальная отмена |

**Процесс:**

1. AI вызывает `cancel_download(id=5)` — возвращает **предпросмотр** без изменений.
2. AI показывает вам название и спрашивает подтверждение.
3. Вы подтверждаете → AI вызывает `cancel_download(id=5, confirm=true)`.

**Примеры:**
```
Отмени загрузку №5
Останови текущую загрузку
Убери из очереди загрузку №3
```

---

#### `delete_download` — Удалить загрузку

Удаляет запись из истории и опционально — файлы с диска. Требует подтверждения.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `id` | int | — | ID загрузки |
| `delete_files` | bool | `true` | Удалить файлы с диска вместе с записью |
| `confirm` | bool | `false` | `false` — предпросмотр, `true` — реальное удаление |

**Примеры:**
```
Удали загрузку №5 со всеми файлами
Удали запись №10 из истории, но оставь файлы
Очисти загрузку №3
```

> Это **необратимая операция** — AI всегда покажет предпросмотр и спросит подтверждение.

## 9. Логи и диагностика

Логи приложения хранятся в:

```
%APPDATA%\YTDownloader\logs\
├── app.log          ← все события уровня INFO и выше
└── app.error.log    ← только ошибки (ERROR и CRITICAL)
```

Если приложение ведёт себя неожиданно:

1. Откройте `app.error.log` в любом текстовом редакторе.
2. Найдите записи с меткой `ERROR` или `CRITICAL` вблизи времени проблемы.
3. При обращении в поддержку приложите содержимое `app.error.log`.

> Логи **не содержат** названий файлов, путей пользователя, URL и прокси-адресов —
> только технические статусы и метрики.

## 10. Частые вопросы

**Приложение скачало один видеоролик вместо всего плейлиста**

Убедитесь, что в ссылке нет параметра `watch?v=...`. Для плейлиста используйте:
`https://www.youtube.com/playlist?list=XXXXXXXXXX`

---

**Видео скачалось, но нет звука / нет изображения**

Это маловероятно — FFmpeg включён в поставку и используется автоматически.
Проверьте `app.error.log` на наличие ошибок с тегом `ffmpeg`.

---

**Не удаётся подключить MCP в Claude или VS Code**

1. Убедитесь, что YTDownloader запущен.
2. Проверьте порт: откройте `http://localhost:8765/mcp` в браузере.
   Должен вернуться JSON-ответ.
3. Если порт изменён в настройках — обновите URL в конфигурации клиента.

---

**Где хранятся скаченные файлы?**

По умолчанию в `%USERPROFILE%\Downloads\YTDownloader\`.
Папку можно изменить в настройках (⚙ → «Выходная папка»).

---

**Можно ли запустить несколько загрузок одновременно?**

Да. Параметр **«Параллельность»** в настройках управляет количеством
одновременных загрузок (по умолчанию — 1, максимум зависит от пропускной способности сети).

---

**Субтитры не скачиваются**

Убедитесь, что:
1. Флаг «Субтитры» включён до добавления ссылки.
2. Указан правильный код языка (например `ru`, а не `russian`).
3. Субтитры действительно существуют для этого видео на YouTube.
